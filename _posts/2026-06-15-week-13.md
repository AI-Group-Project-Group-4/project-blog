## PyTorcs

To do this efficiently, we have taken our Parallel Environment back into the garage and have transformed it into our custom training framework PyTorcs.

PyTorcs builds upon our work on the Parallel Environment taking generalising out the core functions of: Configuration, Training, and Evaluation.

### Configuration

To move away from hard coding parameters into a training script, we moved to a class based configuration system.
A user creates a config file, which details the following:
* Run Name
* Reward Function
* Total Timesteps
* Model Parameters
* Environment Parameters
* Number of Environments

This approach streamlines the experimentation process and collaboration between team members by creating an easy and reproducible file to share.


Below is an example of a config file.

```python
from pytorcs.configs import SACRunConfig
from experiments.SAC.reward_functions import corkscrew_reward

config = SACRunConfig(
    run_name="SAC-Run-1",
    reward_function=corkscrew_reward,
    total_timesteps=10_000_000,
    sac_kwargs={
        "policy":          "MlpPolicy",
        "buffer_size":     50_000,
        "batch_size":      256,
        "train_freq":      2,
        "gradient_steps":  2,
        "learning_rate": 1e-4,
        "ent_coef":      0.05,
        "tau":             0.005,
    },
    env_kwargs={
        "sensor_features": [
            "speedX", "speedY", "speedZ",
            "angle", "trackPos",
            "track",
            "rpm", "gear",
            "wheelSpinVel",
        ],
        "truncate_limit": 10_000,
        "frame_skip":     4,
    },
    num_envs=4,
    base_port=3001,
)
```

### Training and Evaluation

Both training and evaluation are now handled by the scripts `train.py` and `evaluate.py`.
These scripts parse the config files and command arguments directly, and make calls to `pytorcs` backend to load configuration into models and initialise environments and TORCS Docker containers. This process removes the friction we face during rapid iteration, allowing us to focus purely on model development.
